# 提示工程简介 (Introduction)

**提示工程 (Prompt Engineering)** 是一门专注于开发和优化提示词（Prompts）的新兴学科，旨在帮助用户在各种应用场景中高效地利用大语言模型（LLMs）进行构建与应用。

掌握提示工程不仅能提高生产力，还能让我们更深入地理解 LLMs 的能力边界与局限性。
- **研究人员**：利用提示工程在问答、算术推理等各类常规及复杂任务中，显著提升模型的能力并增强其安全性。
- **开发人员**：通过提示工程设计出更稳健、高效的提示技术，从而实现 LLMs 与其他工具及系统的完美对接。

本指南提供了一份全面的参考资料，涵盖了提示工程的理论基础与实战技巧，旨在通过最佳的交互策略，帮助你充分发挥 LLMs 的潜力。

## 实验环境配置

除非另有说明，本指南中的所有示例均在 [OpenAI Playground](https://platform.openai.com/playground) 中使用 `gpt-3.5-turbo` 模型测试完成。模型采用了默认配置项：

```text
模型型号: gpt-3.5-turbo
温度系数 (Temperature): 1
核采样 (Top-P): 1
```

这些提示词策略原则上同样适用于其他具备同等能力的模型，不过实际生成的响应结果可能会因模型差异而有所不同。

## 💡 核心总结 (Core Summary)
- **核心定义**：提示工程是研究如何**开发与优化提示词**以高效应用 LLMs 的学科。
- **双重价值**：既能提升**生产力**，又能帮助我们摸清模型的**能力边界**。
- **应用分工**：
    - **研究者**：关注模型的**安全性**与复杂逻辑推理能力。
    - **开发者**：关注如何设计**稳健的接口**与其内部系统对接。
